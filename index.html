Dicas Gerais para Desenvolvedores

  • Usar nomenclaturas como padrão para nomear arquivos e propriedades em geral, como é o caso do kebab-case e do camelCase.
  • Sempre se manter atualizado para possíveis mudanças na documentação oficial das tecnologias trabalhadas.
  • Escreva um código limpo que seja legível e bem descritivo.
  • Utilizar controle de versão Git.
  • Gerencie seu tempo para que não fique muito tempo preso em um mesmo problema. Uma boa prática é usar a técnica Pomodoro.
  • Faça projetos pessoais.
  • Estudar as tecnologias mais atuais e requisitadas no mercado.

Comandos Git Básicos

git init
git add .
git status
git commit -m "Primeiro commit"
git log
git clone + URL
git config --global user.email
git config --global user.name

Criar uma Ramificação

git checkout -b nome-da-branch
git branch
git merge nome-da-branch
git push origin nome-da-branch

Enviar Projeto para o GitHub

git remote add origin URL
git push -u origin main
